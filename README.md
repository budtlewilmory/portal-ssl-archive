# portal-ssl-archive

本仓库用于归档和发布一组独立的 HTML 页面。这些页面不针对任何特定域名或网站，仅作为通用页面存档使用。

## 项目简介

portal-ssl-archive 是一个静态页面归档仓库，主要用于保存和展示若干独立编写的 HTML 文件。仓库本身不包含后端逻辑，也不绑定任何在线服务或第三方资源。所有页面均可直接通过浏览器打开，适合用于历史留存、离线查看或简单页面分发。

## 目录说明

```
portal-ssl-archive/
├── index.html          # 首页入口
├── pages/              # 存放各独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 静态资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `index.html`：仓库首页，可用于导航或展示说明。
- `pages/`：所有独立 HTML 页面存放于此，文件名对应页面标识。
- `assets/`：页面所需的样式、脚本、图片等资源文件。

## 页面归档说明

- 每个 HTML 页面均为自包含或通过相对路径引用 `assets/` 目录下的资源。
- 页面之间无严格依赖关系，可单独打开使用。
- 归档内容不涉及动态数据、后端接口或外部链接（除非页面本身需要）。
- 所有页面均为静态内容，不收集用户信息，不包含跟踪代码。

## 维护说明

- 本仓库仅用于归档目的，不承诺定期更新或功能迭代。
- 如需新增页面，请将 HTML 文件放入 `pages/` 目录，并按需更新 `index.html` 中的链接。
- 资源文件请统一放入 `assets/` 下对应子目录，避免路径混乱。
- 欢迎提交 Issue 或 Pull Request，但请确保改动不破坏现有页面结构。
- 仓库维护者保留对内容进行整理、删除或归档调整的权利。

## 许可

本仓库内容按需提供，不附加特定许可证。如需引用或分发，请自行评估合规性。
